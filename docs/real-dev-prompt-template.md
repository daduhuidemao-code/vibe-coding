## 开发模式选择指南

### 前后端并行开发

**核心思想**：先定义 API 契约，然后前后端同时开发，最后集成联调。

**适用场景**：
- 团队规模较大（前后端人员分开）
- 项目复杂度较高
- 需要快速迭代

**关键步骤**：
1. 定义 API 契约（请求/响应格式、状态码）
2. 后端实现 API（可以先返回 Mock 数据）
3. 前端使用 Mock 数据开发界面
4. 前后端联调，替换为真实 API
5. 集成测试验证

### TDD 测试驱动开发

**核心思想**：测试先行，用测试驱动开发。

**适用场景**：
- 对代码质量要求高
- 核心业务逻辑复杂
- 需要长期维护的项目

**关键步骤**：
1. 编写失败的测试用例（红色）
2. 实现最小代码使测试通过（绿色）
3. 重构优化代码（保持绿色）
4. 重复以上步骤

### 混合模式（推荐）

在实际项目中，通常采用混合模式：
- **核心模块**（认证、支付、核心业务）：使用 TDD
- **界面层**（UI 组件、页面）：使用标准流程
- **前后端协作**：使用并行开发模式

## 使用指南

### 如何使用这份文档

1. **按阶段选择**：根据当前项目所处阶段，选择对应的提示词模板
2. **填写占位符**：将模板中的 `[占位符]` 替换为实际内容
3. **复制使用**：直接复制到 AI 对话中使用
4. **灵活调整**：根据项目实际情况修改模板内容

### 核心原则

- **意图驱动**：告诉 AI "做什么"，让它决定"怎么做"
- **上下文完整**：提供足够的背景信息和约束条件
- **明确边界**：清楚告诉 AI 什么能做、什么不能做
- **渐进式迭代**：从简单开始，逐步完善

### 真实开发流程要点

1. **不是线性的**：很多阶段是并行进行的（如测试和开发）
2. **迭代进行**：核心开发阶段通常会迭代多次
3. **持续集成**：Git 协作和 CI/CD 贯穿始终
4. **用户反馈驱动**：运营迭代阶段持续收集反馈并改进

这份文档可以作为你进行 Vibe Coding 时的"提示词工具箱"，覆盖真实项目开发的完整生命周期。
